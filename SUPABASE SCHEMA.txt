CORE TABLES
================================================================================

1. PROFILES TABLE
   - Primary user profile information
   - Links to Supabase Auth users
   
   Columns:
   - id (UUID, PRIMARY KEY) - Supabase auth user ID
   - username (TEXT, NOT NULL, UNIQUE) - User's display name
   - email (TEXT, NOT NULL, UNIQUE) - User's email address
   - avatar_url (TEXT) - URL to user's avatar image
   - profession (TEXT) - User's profession
   - trader_status (TEXT) - Current trading status
   - trader_type (TEXT) - Type of trader (e.g., "Day Trader", "Swing Trader")
   - bio (TEXT) - User's biography
   - years_experience (TEXT) - Years of trading experience
   - trading_frequency (TEXT) - How often they trade
   - markets (TEXT) - Markets they trade in
   - trading_goal (TEXT) - Their trading goals
   - trading_challenges (TEXT) - Challenges they face
   - created_at (TIMESTAMP WITH TIME ZONE, NOT NULL, DEFAULT NOW())
   - updated_at (TIMESTAMP WITH TIME ZONE, NOT NULL, DEFAULT NOW())

2. TRADES TABLE
   - Individual trade records
   - Links to user profiles
   
   Columns:
   - id (TEXT, PRIMARY KEY) - Unique trade identifier
   - user_id (TEXT, NOT NULL, REFERENCES profiles(id)) - User who made the trade
   - currency_pair (TEXT, NOT NULL) - Currency pair traded (e.g., "EURUSD")
   - trade_type (TEXT, NOT NULL) - "LONG" or "SHORT"
   - entry_price (DOUBLE PRECISION, NOT NULL) - Entry price
   - exit_price (DOUBLE PRECISION, NOT NULL) - Exit price
   - entry_time (TIMESTAMP, NOT NULL) - When trade was opened
   - exit_time (TIMESTAMP, NOT NULL) - When trade was closed
   - duration (INTEGER, NOT NULL) - Duration in minutes
   - pips (DOUBLE PRECISION, NOT NULL) - Pips gained/lost
   - lot_size (DOUBLE PRECISION, NOT NULL) - Size of the trade
   - profit_loss (DOUBLE PRECISION, NOT NULL) - Profit/loss amount
   - currency (TEXT, NOT NULL, DEFAULT "AUD") - Currency for P&L
   - date (TIMESTAMP, NOT NULL, DEFAULT NOW()) - Trade date
   - notes (TEXT) - Additional notes
   - created_at (TIMESTAMP, NOT NULL, DEFAULT NOW()) - Record creation time
   - tags (TEXT) - Optional tags (max 20 chars, alphanumeric)

3. TRADE_SETUPS TABLE
   - Social trade setup posts
   - Users can share their trading setups with the community
   
   Columns:

   - user_id (UUID, NOT NULL, REFERENCES profiles(id) ON DELETE CASCADE)

   - description (TEXT)

   - dislikes_count (INTEGER, DEFAULT 0)
   - comments_count (INTEGER, DEFAULT 0)
   - created_at (TIMESTAMP WITH TIME ZONE, DEFAULT now())
   - updated_at (TIMESTAMP WITH TIME ZONE, DEFAULT now())

   - Comments on trade setups
   - Supports nested replies
   
   Columns:
   - id (UUID, PRIMARY KEY, DEFAULT gen_random_uuid())
   - trade_setup_id (UUID, NOT NULL, REFERENCES trade_setups(id) ON DELETE CASCADE)
   - user_id (UUID, NOT NULL, REFERENCES profiles(id) ON DELETE CASCADE)
   - content (TEXT, NOT NULL)
   - parent_id (UUID, REFERENCES trade_setup_comments(id) ON DELETE CASCADE)

   - user_id (UUID, NOT NULL, REFERENCES profiles(id) ON DELETE CASCADE)

   Columns:

   - user_id (UUID, NOT NULL, REFERENCES profiles(id) ON DELETE CASCADE)

   - UNIQUE(trade_setup_id, user_id)

7. TRADE_SETUP_COMMENT_REACTIONS TABLE

   Columns:

   - user_id (UUID, NOT NULL, REFERENCES profiles(id) ON DELETE CASCADE)

   - updated_at (TIMESTAMP WITH TIME ZONE, DEFAULT now())

   - Cleans up trade setup and all related data
   - Removes comments, likes, tags, and the setup itself

8. get_user_conversations(user_uuid UUID)
   - Returns list of user's conversations with metadata

9. soft_delete_message(message_uuid UUID, user_uuid UUID)
   - Soft deletes a message (sets deleted_at timestamp)

10. cleanup_old_messages()

11. get_comment_with_reactions(comment_id UUID)

================================================================================
ROW LEVEL SECURITY (RLS) POLICIES
================================================================================

All tables have RLS enabled with appropriate policies:

1. PROFILES
   - Users can view public profiles
   - Users can update their own profile
   - Users can delete their own profile

2. TRADES
   - Users can view their own trades
   - Users can insert/update/delete their own trades

3. TRADE_SETUPS
   - Users can view public setups or their own
   - Users can insert/update/delete their own setups

4. TRADE_SETUP_COMMENTS
   - Users can view comments on setups they can see
   - Users can insert comments on setups they can see
   - Users can update/delete their own comments

5. TRADE_SETUP_LIKES/DISLIKES
   - Users can view likes/dislikes on setups they can see
   - Users can add/remove their own likes/dislikes

6. MESSAGES
   - Users can only see messages they sent or received
   - Users can only send messages as themselves
   - Users can update/delete their own messages

7. FRIEND_REQUESTS
   - Users can view requests they sent or received
   - Users can send requests to other users
   - Users can update/delete their own requests

8. TRADER_FRIENDS
   - Users can view their own friend relationships
   - Users can manage their own friend relationships

9. USER_BLOCKS
   - Users can view their own blocks
   - Users can add/remove their own blocks

10. USER_SETTINGS
    - Users can only access their own settings

11. USER_PRESENCE
    - Users can view presence of users they can see
    - Users can update their own presence
