üóÑÔ∏è Database Setup

Supabase Configuration

  1. Create a new Supabase project
  2. Enable the required extensions (pgcrypto, uuid-ossp)
  3. Run the database migrations
  4. Configure Row Level Security (RLS) policies
  5. Set up storage buckets for file uploads

Database Schema

The application uses the following main tables:

  • profiles: User profile information
  • trades: Trade records and analysis
  • trade_setups: Top-down analysis setups
  • friends: User connections and relationships
  • messages: Direct messaging system
  • notifications: User notifications

üîå API Endpoints

Authentication

  • POST /api/auth/login - User login
  • POST /api/auth/register - User registration
  • POST /api/auth/logout - User logout

Trades

  • GET /api/trades - Fetch user trades
  • POST /api/trades - Create new trade
  • PUT /api/trades/[id] - Update trade
  • DELETE /api/trades/[id] - Delete trade

Social Features

  • GET /api/traders - Fetch traders list
  • POST /api/social/trade-setup - Share trade setup
  • POST /api/notifications/friend-request - Send friend request

üîß Troubleshooting

Common Issues

Database Connection Issues

Problem: Unable to connect to Supabase database

Solution: Verify your environment variables and check Supabase project status

Authentication Problems

Problem: Users unable to login or register

Solution: Check Supabase Auth settings and email configuration

File Upload Issues

Problem: Images or files not uploading

Solution: Verify Supabase Storage bucket permissions and policies

Performance Optimization

  • Enable Supabase connection pooling for production
  • Use Next.js Image optimization for better loading
  • Implement proper caching strategies
  • Monitor database query performance

Security Best Practices

  • Always use Row Level Security (RLS) in Supabase
  • Validate all user inputs with Zod schemas
  • Implement proper error handling
  • Regular security audits and updates
